## 1. ภาพรวมโครงการ

### 1.1 วัตถุประสงค์
พัฒนา Web Application สำหรับจัดการระบบการเงินของโรงเรียนวัดบ้านดอน สังกัด สพป.ระยอง เขต 1 เพื่อทดแทนการทำงานผ่าน Excel ให้มี Workflow ชัดเจน ติดตามสถานะได้ พิมพ์เอกสารราชการได้ และมี Dashboard รายงานภาพรวม

### 1.2 ขอบเขต
- ระบบเบิกจ่ายเงิน (Disbursement)
- ระบบอนุมัติ (Approval Workflow)
- ระบบบัญชีธนาคาร + ประเภทเงิน
- ระบบรายงานเงินคงเหลือ (Balance Report)
- ระบบหักภาษี ณ ที่จ่าย + ใบ 50 ทวิ
- ระบบกระทบยอด (Reconciliation)
- Dashboard + รายงาน
- พิมพ์เอกสารราชการ (ตราครุฑ)
- รองรับการขยายโมดูลในอนาคต (เช่น ข้อมูลนักเรียน)

### 1.3 ปีงบประมาณ
- เริ่ม **1 ตุลาคม** ถึง **30 กันยายน** ของปีถัดไป
- ใช้ปี พ.ศ. (เช่น ปีงบประมาณ 2569 = 1 ต.ค. 2568 - 30 ก.ย. 2569)

## 6. ระบบเบิกจ่ายเงิน (Disbursement)

### 6.1 โครงสร้างข้อมูล

```
ApprovalRequest (บันทึกขออนุมัติ) — 1 ฉบับ
├── memoNumber: บันทึกฉบับที่ (เช่น "พิเศษ/2569")
├── requestDate: วันที่
├── fiscalYear: ปีงบประมาณ
├── totalAmount: ยอดรวมทั้งสิ้น
├── status: สถานะ Workflow
├── createdBy: ครูการเงินผู้สร้าง
│
└── DisbursementGroup (กลุ่มตามประเภทเงิน) — หลายกลุ่มต่อ 1 ฉบับ
    ├── budgetTypeId → ได้บัญชีธนาคารอัตโนมัติ
    ├── subtotal: ยอดรวมกลุ่ม
    │
    └── DisbursementItem (รายการย่อย) — หลายรายการต่อ 1 กลุ่ม
        ├── description: รายละเอียด
        ├── amount: จำนวนเงิน
        ├── taxWithheld: หักภาษี ณ ที่จ่าย (ถ้ามี, ไม่บังคับ)
        ├── netAmount: ยอดจ่ายจริง
        ├── payeeName: ชื่อผู้รับจ้าง
        └── note: หมายเหตุ
```

### 6.3 หน้าจอสร้างบันทึกขออนุมัติ

```
1. กรอก วันที่ + บันทึกฉบับที่
2. กด "+ เพิ่มประเภทเงิน"
```

## 7. Workflow (กระบวนการอนุมัติ)

### 7.1 ตั้งค่า Workflow Mode

| โหมด | คำอธิบาย | เหมาะกับ |
|------|---------|---------|
| **ครูการเงินดำเนินการเอง** | ไม่ต้องรออนุมัติ ทำทุกขั้นตอนเอง | เริ่มต้นใช้งาน (Default) |
| **ต้องผ่านการอนุมัติ** | ส่งเรื่องให้ผู้บริหาร อนุมัติก่อน | ระบบสมบูรณ์ |

ตัวเลือกย่อย (ถ้าเปิดระบบอนุมัติ):
- รอง ผอ. เห็นชอบอย่างเดียว
- ผอ. อนุมัติอย่างเดียว
- ทั้ง รอง ผอ. + ผอ. (2 ขั้น)

### 7.2 ขั้นตอน 7 Steps

```
โหมด "ครูการเงินทำเอง":
  1 → 2 → 3 → 4 → (5 ถ้ามีภาษี) → 6 → (7 ถ้ามีภาษี)
  💰   💰   💰   💰      💰          💰       💰
  ครูการเงินทำทุกขั้นตอน

โหมด "ต้องอนุมัติ":
  1 → 2 → [รออนุมัติ] → 3 → 4 → (5) → 6 → (7)
  💰   💰     👔        💰   💰   💰    💰    💰
```

| Step | ชื่อ | Status | ผู้ดำเนินการ |
|------|------|--------|-------------|
| 1 | ยื่นเรื่องของบ | DRAFT | ครูการเงิน |
| 2 | ทำหนังสือขออนุมัติ | PENDING_APPROVAL | ครูการเงินสร้าง |
| (2.5) | อนุมัติ | APPROVED | รอง ผอ./ผอ. (ถ้าเปิดระบบ) |
| 3 | เบิกเงินที่ธนาคาร | WITHDRAWN | ครูการเงิน |
| 4 | นำจ่ายผู้รับจ้าง | PAID | ครูการเงิน |
| 5 | ออกใบ 50 ทวิ | TAX_ISSUED | ครูการเงิน **(ถ้ามีหักภาษีเท่านั้น)** |
| 6 | แจ้งยอดเงินคงเหลือ | BALANCE_REPORTED | ครูการเงิน |
| 7 | บันทึกสรุปภาษี | COMPLETED | ครูการเงิน **(ถ้ามีหักภาษีเท่านั้น)** |

### 7.3 ภาษี ณ ที่จ่าย — ไม่บังคับทุกรายการ

```
- ไม่ใช่ทุกรายการจ่ายจะหักภาษี
- ถ้ามีหักภาษี → ต้องออกใบ 50 ทวิ ให้ผู้รับจ้าง
- ภาษีที่หักไว้ → รวมยอดรายเดือน → นำส่งสรรพากร ภายในวันที่ 7 ของเดือนถัดไป
- ขั้นตอน 5 และ 7 ข้ามได้ถ้าไม่มีภาษี
```

## 8. ระบบเงินเคลื่อนไหว (Money Transaction)

### 8.1 ประเภทการเคลื่อนไหว

| ประเภท | คำอธิบาย | ตัวอย่าง |
|--------|---------|---------|
| IN (เงินเข้า) | เงินโอนเข้าบัญชี | เงินอุดหนุนโอนเข้า, ดอกเบี้ย |
| OUT (เงินออก) | เบิกจ่ายจากบัญชี | จ่ายผู้รับจ้าง (ผูกกับ ApprovalRequest) |

### 8.2 ความสัมพันธ์

```
MoneyTransaction (รายการเคลื่อนไหว)
├── type: "IN" หรือ "OUT"
├── bankAccountId: บัญชีธนาคาร
├── budgetTypeId: ประเภทเงิน
├── amount: จำนวนเงิน
├── description: รายละเอียด
├── transactionDate: วันที่
├── approvalRequestId: ผูกกับบันทึกขออนุมัติ (เฉพาะเงินออก, ถ้ามี)
└── createdBy: ผู้บันทึก

เงินออก → มาจาก ApprovalRequest (อัตโนมัติเมื่อเบิกจ่าย)
เงินเข้า → ครูการเงินบันทึกเอง (เงินอุดหนุนโอนเข้า)
```

## 9. รายงานเงินคงเหลือ (Balance Report)

### 9.1 หลักการ
- **ทำเฉพาะวันที่มีเงินเคลื่อนไหว** (เข้าหรือออก)
- วันที่ไม่มีการเคลื่อนไหว ไม่ต้องทำรายงาน
- โครงสร้างรายการส่วนใหญ่ **Fix** (คงที่) ตามที่ Admin ตั้งค่า
- สามารถเพิ่ม/ตัดรายการได้เมื่อจำเป็น

### 9.2 คอลัมน์

| คอลัมน์ | คำอธิบาย |
|---------|---------|
| ประเภทเงินคงเหลือ | ชื่อรายการ |
| เงินสด | ยอดเงินสด |
| จำนวนเงินฝากธนาคาร | ยอดในบัญชีธนาคาร |
| จำนวนเงินสะสมรวมยังไม่ได้ | ยอดสะสม |
| รวม | ผลรวม |
| หมายเหตุ | เปลี่ยนแปลง (+/-) |

### 9.3 ลายเซ็น (3 ชุด)
1. ครูการเงิน (เจ้าหน้าที่การเงิน) + ผู้รายงาน
2. คณะกรรมการเก็บรักษาเงิน (4 คน)
3. รอง ผอ. + ผอ.

### 9.4 ที่มาของตัวเลข
- ยอดคำนวณจาก MoneyTransaction สะสม (เงินเข้า - เงินออก)
- หมายเหตุ = ยอดเปลี่ยนแปลงของวันนั้น

## 10. ระบบกระทบยอด (Reconciliation)

### 10.1 วัตถุประสงค์
Double check — เทียบยอดในระบบกับยอดจริงจากธนาคาร

### 10.2 หลักการ
- **กระทบเฉพาะบัญชีที่มีเงินเคลื่อนไหวในวันนั้น** (ไม่ต้องกรอกทุกบัญชี)
- ระบบดึงบัญชีที่มีรายการเคลื่อนไหวอัตโนมัติ
- ครูการเงินกรอกยอดจากธนาคาร (Statement/สมุดบัญชี)
- ระบบเทียบ: ยอดในระบบ vs ยอดจากธนาคาร

### 10.3 ผลลัพธ์

| สถานะ | เงื่อนไข | การดำเนินการ |
|-------|---------|-------------|
| ✅ ตรงกัน | ผลต่าง = 0 | บันทึกสำเร็จ |
| ❌ ไม่ตรง | ผลต่าง ≠ 0 | ต้องกรอกหมายเหตุ อธิบายสาเหตุ |

### 10.4 สิทธิ์

| การกระทำ | ครูการเงิน | Admin | รอง ผอ./ผอ. |
|---------|:---------:|:-----:|:-----------:|
| กระทบยอด (กรอก) | ✅ | ✅ | ❌ |
| ดูผลกระทบยอด | ✅ | ✅ | ✅ |

## 14. Database Schema (สรุป)

```
User               — ผู้ใช้งาน (5 roles)
Session            — Cookie sessions
Permission         — สิทธิ์ (group + action)
RolePermission     — Role ↔ Permission mapping (Admin ปรับได้)

BankAccount        — บัญชีธนาคาร
BudgetType         — ประเภทเงิน (ผูก BankAccount, self-ref parentId)

ApprovalRequest    — บันทึกขออนุมัติ (Workflow 7 steps)
DisbursementGroup  — กลุ่มรายการตามประเภทเงิน (ต่อ 1 ApprovalRequest)
DisbursementItem   — รายการย่อย (ต่อ 1 Group)
WorkflowAction     — Audit trail (ใครทำอะไรเมื่อไหร่)

MoneyTransaction   — เงินเข้า/ออก (ทุกรายการเคลื่อนไหว)
TaxCertificate     — ใบ 50 ทวิ (ผูก DisbursementItem, ถ้ามี)
TaxPaymentSummary  — สรุปนำส่งภาษีรายเดือน

BalanceReport      — รายงานคงเหลือ (snapshot ณ วันที่)
BalanceEntry       — รายการย่อยในรายงาน (ต่อ BudgetType)

Reconciliation     — กระทบยอด (ต่อวัน ต่อบัญชี)
WorkflowSetting    — ตั้งค่า Workflow mode
```
